4. WebRTC-Medienübertragung

Audio und Video werden mittels WebRTC (Web Real-Time Communication) übertragen — einem offenen Standard, der in moderne Browser integriert ist. Medien fließen durch unseren SFU-Server (Selective Forwarding Unit), der verschlüsselte Medienströme zwischen Teilnehmern weiterleitet, ohne sie zu entschlüsseln oder zu speichern.

Zur NAT-Traversierung (Verbindung von Teilnehmern hinter Firewalls oder Routern) kann der Dienst TURN-Relay-Server verwenden. In diesem Fall wird Ihr Mediendatenverkehr in verschlüsselter Form über den TURN-Server weitergeleitet. Die verwendeten TURN-Server sind: [LISTE DER TURN-SERVER ODER „eigene Infrastruktur"].

Die gesamte Signalisierung (Sitzungsaushandlung) wird über eine verschlüsselte WebSocket (WSS)-Verbindung mit TLS übertragen.

5. Lokale Datenspeicherung (nur auf Ihrem Gerät)

Die folgenden Daten werden im localStorage Ihres Browsers unter dem Schlüssel edumeetRoot gespeichert. Diese Daten verlassen Ihr Gerät nie und sind für uns nicht zugänglich.

  • Anzeigename (damit Sie ihn nicht bei jedem Besuch erneut eingeben müssen)
  • Ausgewähltes Audioeingabegerät
  • Ausgewähltes Audioausgabegerät
  • Ausgewähltes Videogerät
  • Audioqualitätseinstellungen (Rauschunterdrückung, Echounterdrückung, Codec-Präferenzen)
  • Videoqualitätseinstellungen (Auflösung, Bildrate)
  • Sprache der Benutzeroberfläche (Locale)
  • UI-Präferenzen (Layout, Eigenansicht spiegeln, max. sichtbare Videos)

Sie können diese Daten jederzeit über Ihre Browsereinstellungen löschen (Websitedaten löschen / localStorage löschen).

6. Lokale Aufzeichnung (optionale Funktion)

Die Plattform bietet eine optionale Funktion zur lokalen Aufzeichnung. Bei deren Nutzung gilt:

  • Die Aufzeichnung erfolgt vollständig in Ihrem Browser über die MediaRecorder-API.
  • Die Aufzeichnungsdatei wird direkt an einem von Ihnen über einen Speicherdialog gewählten Ort auf Ihrem eigenen Gerät gespeichert.
  • Aufzeichnungsdaten werden nicht auf unsere Server hochgeladen.
  • Andere Teilnehmer werden benachrichtigt, wenn eine Aufzeichnung aktiv ist.
  • Wenn Sie eine Sitzung aufzeichnen, werden Sie zum Verantwortlichen für diese Aufzeichnung und sind für deren rechtmäßige Verwendung verantwortlich.

7. Hintergrundeffekte und KI-Verarbeitung

Die Plattform bietet optionale Funktionen zur Hintergrundunschärfe und virtuellen Hintergründen. Diese Verarbeitung:

  • Findet vollständig lokal in Ihrem Browser mit WebGL und TensorFlow Lite statt.
  • Sendet keine Videoframes zur Verarbeitung an externe Server.
  • Von Ihnen hochgeladene benutzerdefinierte Hintergrundbilder werden ausschließlich im localStorage Ihres Browsers gespeichert.

8. Verwaltungs- / Administrationsfunktionen (optional)

Wenn der Betreiber das Verwaltungssystem aktiviert hat, können Raumadministratoren sich mit einer E-Mail-Adresse und einem Passwort anmelden. In diesem Fall werden verarbeitet:

  • E-Mail-Adresse und ein JWT-Authentifizierungstoken.
  • Das Token wird im localStorage unter dem Schlüssel feathers-jwt gespeichert und dient der Sitzungskontinuität.
  • Dies gilt ausschließlich für Administratorkonten, nicht für reguläre Konferenzteilnehmer.

9. Anmeldung über OAuth 2.0 / OpenID Connect (OIDC) (optional)

Wenn der Betreiber die föderierte Anmeldung aktiviert hat, können Benutzer (in der Regel Administratoren oder Mitglieder einer Institution) sich über einen externen Identitätsanbieter (IdP) mittels des Protokolls OAuth 2.0 / OpenID Connect authentifizieren. Dies gilt nur, wenn in der Dienstkonfiguration loginEnabled gesetzt ist — für anonyme Konferenzteilnehmer ist diese Option nicht verfügbar.

Funktionsweise

Wenn Sie auf „Anmelden" klicken, wird Ihr Browser zum Autorisierungsendpunkt des Identitätsanbieters (authorize_url) weitergeleitet. Sie authentifizieren sich dort direkt — Ihr Passwort wird nie an diesen Dienst übermittelt. Nach erfolgreicher Authentifizierung gibt der Identitätsanbieter einen Autorisierungscode zurück, den der Server am access_url des Anbieters gegen ein Token eintauscht. Ihre Profildaten werden anschließend vom profile_url des Anbieters abgerufen.

Vom Identitätsanbieter empfangene Daten

Der standardmäßig angeforderte OAuth-Scope ist openid profile email. Die folgenden Claims können je nach Rückgabe des Identitätsanbieters empfangen werden:

Token-Speicherung

Nach der Anmeldung wird ein JWT-Token im localStorage Ihres Browsers unter dem Schlüssel feathers-jwt gespeichert. Dieses Token dient der Authentifizierung nachfolgender Anfragen an die Verwaltungs-API, ohne dass eine erneute Anmeldung erforderlich ist. Es wird kurz vor Ablauf automatisch erneuert. Das Token wird beim Abmelden aus dem localStorage entfernt.
